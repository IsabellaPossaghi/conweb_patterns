Once oriented, how can the user navigate the website according to different goals?

3 Patterns

A conversation must enhance orientative processes allowing browsing through user-labelled nodes

A solution extracted from the challenge about enabling the saving of “chosen information”, allowing an intervention on default explorational experiences through the medium of “navigational checkpoints”: a mechanism that enhances orientaional processes by allowing the browsing through user-labelled nodes.

Useful tip:
Add customization options in order to allow the user to customise their choice, enhancing memorability with personalised labelling.

A conversation must support direct probing about the offerings in natural language, directly pose questions to ask whether the website is supplying a specific feature

With the aim of directly inquiring the conversational agent about punctual, fast-served requests, the navigation by question and answer is developed as a suitable solution. A conversation should support direct probing about the offerings in natural language, and posing questions to ask whether the website is supplying a specific feature.

Knowing the complete Website offer, prior to formulating a request, constitutes a big obstacle in navigation. Explicitly formulating state requests even without having fully explored or learnt the whole structure, grants the user the possibility to fulfil specific information needs with a more direct access to the offerings or features. With this mechanism the user can directly pose questions to the bot to ask whether the website is offering a specific feature or narrow down the list of offerings. In this case, nodes are matched against the natural language expression of the information needed.

Useful tip:
Allow the user to search “locally” (for example, at single-page level) as well as on global level (the entire website). This mechanism can be fed even by an external navigation context, for example a question when coming from a search engine.

A conversation must support vertical navigation to explore layers in depth until a desired content is reached

In order to support users’ challenge to achieve a desired information by in-depth exploration of a chosen area, conversation should bolster vertical navigation to explore layers in depth until a desired content is reached.
This navigational model is intended to assist users in exploring the content hierarchy employing the multi-layered navigation to also speed up the learning process.

Useful tip:
Add option provision on request allowing the user to choose between a limited number of options, making the remaining available if demanded. In combination with this suggestion, a clusterization content arrangement according to “grouping and nesting” is advised.
